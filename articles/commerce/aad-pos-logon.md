<span data-ttu-id="509d7-105">Einzelhändler, die Dynamics 365 Commerce zusammen mit anderen Microsoft Cloud-Diensten wie Microsoft Azure, Microsoft 365 und Microsoft Teams verwenden, möchten in der Regel Azure AD für die zentrale Verwaltung von Anmeldeinformationen für eine sichere und nahtlose Anmeldung in allen Anwendungen verwenden.</span><span class="sxs-lookup"><span data-stu-id="509d7-105">Retailers who use Dynamics 365 Commerce along with other Microsoft cloud services such as Microsoft Azure, Microsoft 365, and Microsoft Teams typically want to use Azure AD for centralized management of user credentials for a secure and seamless sign-in experience across applications.</span></span> <span data-ttu-id="509d7-106">Um die Azure AD-Authentifizierung für Commerce POS zu verwenden, müssen Sie zuerst Azure AD als Authentifizierungsmethode in der Commerce-Zentrale konfigurieren.</span><span class="sxs-lookup"><span data-stu-id="509d7-106">To use Azure AD authentication for Commerce POS, you must first configure Azure AD as the authentication method in Commerce headquarters.</span></span>

## <a name="configure-pos-authentication-method"></a><span data-ttu-id="509d7-107">Konfigurieren Sie die POS-Authentifizierungsmethode</span><span class="sxs-lookup"><span data-stu-id="509d7-107">Configure POS authentication method</span></span>

<span data-ttu-id="509d7-108">Um die POS-Authentifizierungsmethode in der Commerce-Zentrale zu konfigurieren, führen Sie diese Schritte aus.</span><span class="sxs-lookup"><span data-stu-id="509d7-108">To configure the POS authentication method in Commerce headquarters, follow these steps.</span></span>
    
1. <span data-ttu-id="509d7-109">Gehen Sie zu **Retail und Commerce \> Kanalkonfiguration \> POS-Konfiguration \> POS-Profile \> Funktionalitätsprofile** und wählen Sie ein Funktionalitätsprofil zum Ändern.</span><span class="sxs-lookup"><span data-stu-id="509d7-109">Go to **Retail and Commerce \> Channel setup \> POS setup \> POS profiles \> Functionality profiles**, and select a functionality profile to change.</span></span>
1. <span data-ttu-id="509d7-110">Wählen Sie im Abschnitt **Anmeldung von POS-Mitarbeitern** auf der Registerkarte **Funktionen** eine gewünschte Option für die Authentifizierungsmethode aus der Dropdown-Liste **Authentifizierungsmethode für die Anmeldung**.</span><span class="sxs-lookup"><span data-stu-id="509d7-110">In the **POS staff logon** section of the **Functions** FastTab, select a desired authentication method option from the **Logon authentication method** drop-down list.</span></span>

    <span data-ttu-id="509d7-111">Die **Anmelde-Authentifizierungsmethode** hat drei Optionen:</span><span class="sxs-lookup"><span data-stu-id="509d7-111">The **Logon authentication method** has three options:</span></span>
    
    - <span data-ttu-id="509d7-112">**Personal-ID und Kennwort** – Diese Standardoption erfordert, dass POS-Benutzer eine Personal-ID und ein Kennwort eingeben, um sich am POS anzumelden und um auf die Manager-Überschreibungsfunktion zuzugreifen.</span><span class="sxs-lookup"><span data-stu-id="509d7-112">**Personnel ID and Password** - This default option requires POS users to enter a personnel ID and password to sign in to the POS and to access manager override functionality.</span></span>
    - <span data-ttu-id="509d7-113">**Azure AD ohne Single Sign-On** – Bei dieser Option müssen POS-Benutzer Azure AD-Anmeldeinformationen verwenden, um sich an der Kasse anzumelden und auf die Manager-Override-Funktionalität zuzugreifen.</span><span class="sxs-lookup"><span data-stu-id="509d7-113">**Azure AD without single sign-on** - This option requires POS users to use Azure AD credentials to sign in to the POS and access manager override functionality.</span></span> <span data-ttu-id="509d7-114">Wenn der POS Client aktualisiert oder neu geöffnet wird, muss der POS-Benutzer Anmeldeinformationen Azure AD angeben, um sich erneut anzumelden.</span><span class="sxs-lookup"><span data-stu-id="509d7-114">When the POS client is refreshed or reopened, the POS user must provide Azure AD credentials to sign in again.</span></span>
    - <span data-ttu-id="509d7-115">**Azure AD mit Single Sign-On** – Wenn diese Option ausgewählt ist, können sich POS-Benutzer bei Cloud POS (CPOS) mit aktiven Anmeldeinformationen Azure AD anmelden, die von anderen Webanwendungen im selben Webbrowser verwendet werden, oder sich bei Modern POS (MPOS) mit Anmeldeinformationen Azure AD anmelden, die bei Windows angemeldet sind.</span><span class="sxs-lookup"><span data-stu-id="509d7-115">**Azure AD with single sign-on** - When this option is selected, POS users are able to sign in to Cloud POS (CPOS) using active Azure AD credentials that are being used by other web applications in the same web browser, or sign in to Modern POS (MPOS) using Azure AD credentials signed in to Windows.</span></span> <span data-ttu-id="509d7-116">Beide Methoden lassen die Anmeldung zu, ohne dass die Anmeldeinformationen Azure AD auf dem POS-Anmeldebildschirm eingegeben werden müssen.</span><span class="sxs-lookup"><span data-stu-id="509d7-116">Both methods allow sign-in without needing to enter Azure AD credentials on the POS sign-in screen.</span></span> <span data-ttu-id="509d7-117">Für den Zugriff auf die Überschreibungsfunktion des POS-Managers ist jedoch immer noch eine Anmeldung mit Anmeldeinformationen Azure AD erforderlich.</span><span class="sxs-lookup"><span data-stu-id="509d7-117">However, accessing the POS manager override functionality will still require sign-in using Azure AD credentials.</span></span>

1. <span data-ttu-id="509d7-118">Gehen Sie zu **Retail und Commerce > Retail und Commerce IT > Verteilungsplan** und führen Sie den Job **1070 (Kanalkonfiguration)** aus, um die neuesten Einstellungen des Funktionalitätsprofils mit den POS Clients zu synchronisieren.</span><span class="sxs-lookup"><span data-stu-id="509d7-118">Go to **Retail and Commerce > Retail and Commerce IT > Distribution schedule** and run the **1070 (Channel configuration)** job to synchronize the latest functionality profile settings to POS clients.</span></span>

> [!NOTE]

<span data-ttu-id="509d7-129">Sie müssen den Job **1060 (Mitarbeiter)** in **Retail und Commerce > Retail und Commerce IT > Verteilungsplan** ausführen, um die neuesten POS-Benutzer- und Azure AD-Kontodaten mit dem Kanal zu synchronisieren.</span><span class="sxs-lookup"><span data-stu-id="509d7-129">You must run the **1060 (Staff)** job in **Retail and Commerce > Retail and Commerce IT > Distribution schedule** to synchronize the latest POS user and Azure AD account data to the channel.</span></span>
