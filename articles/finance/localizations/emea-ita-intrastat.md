Sie müssen zwei Meldungen an die Behörde senden. Eine Meldung gilt für innergemeinschaftliche Versendungen und die andere für innergemeinschaftliche Ankünfte. Jeder Bericht besteht aus fünf Teilen:

- **Deckblatt**

- **Abschnitt 1. Waren**: Dieser Abschnitt enthält Informationen zu normalen Transaktionen und Gutschriften, die sich auf Rechnungen für Waren im gleichen Berichtszeitraum wie der Intrastat-Meldungszeitraum beziehen.
- **Abschnitt 2. Korrekturen für Waren**: Dieser Abschnitt enthält Informationen zu Korrekturen und Gutschriften, die sich auf Rechnungen für Waren im vorherigen Berichtszeitraum wie der Intrastat-Meldungszeitraum beziehen.
- **Abschnitt 3. Dienstleistungen**: Dieser Abschnitt enthält Informationen zu normalen Transaktionen und Gutschriften, die sich auf Rechnungen für Dienstleistungen im gleichen Berichtszeitraum wie der Intrastat-Meldungszeitraum beziehen.
- **Abschnitt 4. Korrekturen für Dienstleistungen**: Dieser Abschnitt enthält Informationen zu Korrekturen und Gutschriften, die sich auf Rechnungen für Dienstleistungen im vorherigen Berichtszeitraum wie der Intrastat-Meldungszeitraum beziehen.

## <a name="set-up-intrastat"></a>Einrichten von Intrastat

### <a name="preliminary-setup"></a>Vorläufige Einstellungen

Die folgenden allgemeinen Informationen sollten vor Beginn der Arbeit mit Intrastat eingerichtet werden:

   - Warencodes. Für Dienstleistungen sollten Sie sechsstellige Warennummern definieren.
   - Art des Geschäftes. Beachten Sie, dass Italien einstellige Transaktionscodes verwendet.
   - Transportmethoden.
   - Statistikverfahren.
   - Außenhandelsparameter.
   - Lagerort.
   - Details für freigegebene Produkte.
   - Kontaktinformationen des Beauftragten.

Weitere Informationen finden Sie unter [Intrastat Übersicht](emea-intrastat.md).

### <a name="set-up-italian-intrastat"></a>Einrichten von Intrastat für Italien

Befolgen Sie diese Schritte, um italienisch-spezifische Optionen einzurichten, damit Sie mit Intrastat arbeiten können.

1. In  [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/Logon/Index), in der Bibliothek der freigegebenen Anlagen laden Sie die neuesten Versionen der Konfigurationen für die elektronische Berichterstattung (ER) für das Format der Intrastaterklärung herunter:

    - Intrastat-Modell
    - Intrastat-Bericht
    - Intrastat (IT)

    Weitere Informationen finden Sie unter [Elektronische Berichtskonfigurationen aus Lifecycle Services herunterladen](../../fin-ops-core/dev-itpro/analytics/download-electronic-reporting-configuration-lcs.md).

2. Gehen Sie in Dynamics 365 Finance zu **Steuern** > **Einrichtung** > **Außenhandelsparameter**.
3. Auf der **Intrastat** Registerkarte im Inforegsiter **Allgemein** legen Sie die folgenden Felder fest:

    - **Herkunfts-/Bestimmungsland**: Wählen Sie den Landkreis Ihres Unternehmens aus. Dieses Bundesland wird für Sendungen verwendet.
    - **Transaktions-Code**: Wählen Sie den Transaktionscode für Eigentumsübertragungen. Dieser Code wird für Transaktionen verwendet, die eine tatsächliche oder geplante Eigentumsübertragung gegen Entschädigung bewirken, sowie für Korrekturen.

    - **Gutschrift**: Wählen Sie den Transaktionscode für die Warenrücksendung aus.
    - **Umsatzberichtszeitraum**: Wählen Sie den Meldezeitraum für die Ausfuhranmeldung: **Monat** oder **Quartal**. Vierteljährliche Meldungen werden im vereinfachten Format exportiert.
    - **Kaufberichtszeitraum**: Wählen Sie den Meldezeitraum für die Importanmeldung: **Monat** oder **Quartal**. Vierteljährliche Meldungen werden im vereinfachten Format exportiert.

4. Legen Sie auf dem Inforegister **Elektronische Berichterstattung** die folgenden Felder fest:

    - **Dateiformatzuordnung**: **Intrastat (IT)** auswählen.
    - **Berichtformatzuordnung**: **Intrastatbericht** auswählen.

5. Auf dem Inforegister **Warencodehierarchie** im Feld **Kategoriehierarchie** wählen Sie **Intrastat**.
6. Auf dem Inforegsiter **Statistischer Wert** stellen Sie die **Statistische Daten drucken und exportieren** Option auf **Ja**, sofern erforderlich. Diese Einstellung aktiviert die Übertragung des Statistikteils. Der Statistikteil besteht aus Daten zu Gewichten, Zusatzeinheiten, Statistikwerten, Lieferbedingungen, Lieferabrufen, Verkehrsträgern und Herkunftsregionen.

    > [!NOTE]
    > Bei einer vierteljährlichen Meldung enthält der Intrastat-Bericht weder den statistischen Abschnitt noch Informationen zu Lieferbedingungen und Transportarten. Weitere Informationen finden Sie in der Tabelle [Übersicht](#overview) weiter oben in diesem Artikel.

7. Auf der Registerkarte **Länder-/Regionseigenschaften** führen Sie alle Länder oder Regionen auf, mit denen Ihre Organisation Geschäfte macht. Legen Sie für jedes Land oder Region die folgenden Felder fest:

    - **Land/Region der Partei**: Wählen Sie den Länder- bzw. Regionscode aus.
    - **Intrastat-Code**: Geben Sie den zweistelligen Intrastat-Code ein.
    - **Währung**: Geben Sie die Landeswährung des Landes oder der Region an. Wenn der Lieferant seinen Sitz in einem EU-Land hat, das den Euro nicht verwendet, werden die Rechnungsbeträge sowohl in der Währung des Lieferanten als auch in Euro gemeldet. Wenn der Verkäufer beispielsweise in Dänemark ansässig ist, werden die gemeldeten Beträge für die Einfuhranmeldung sowohl in dänischen Kronen (DKK) als auch in Euro (EUR) angegeben.
    - **Land-/Regionstyp**: Wählen Sie den Typ des Landes oder der Region in Bezug auf Ihre Organisation aus. Für das Intrastat-Journal werden nur Länder oder Regionen des **EU** und **Spezielles Inland** Typ übertragen.

    > [!NOTE]
    > Für Länder oder Regionen des Typs **Spezielles Inland** Typ werden die folgenden Felder in der Intrastat-Berichtsdatei ausgelassen: **Gewicht**, **Zusätzliche Einheiten**, **Statistischer Wert**, **Lieferbedingungen**, **Transportcode**, **Herkunftsland/-Region/Zielort**, und **Herkunfts-/Bestimmungsland**. Zum Beispiel im Feld **Land/-Region Partei** wählen Sie **SMR (San Marino)**, und dann im Feld **Land-/Regionstyp** wählen Sie **Spezielles Inland**.

8. Gehen Sie zu **Kreditoren** > **Einrichtung** > **Lieferbedingungen**.
9. Wählen Sie im Raster die Lieferbedingungen aus.
10. Auf dem Inforegister **Allgemein** im **Intrastat-Code** geben Sie den einstelligen Code ein, der im Intrastat-Bericht verwendet wird.
11. Weisen Sie Debitoren und Kreditoren Steuerbefreiungsnummern zu, indem Sie diese Schritte ausführen. Diese Zahlen erscheinen im Intrastat-Bericht.
12. Navigieren Sie zu **Steuer** > **Setup** > **Mehrwertsteuer** > **Steuerbefreite Nummern**, und führen Sie alle steuerbefreiten Nummern für Ihre Debitoren und Kreditoren auf. Legen Sie für jeden Partern die folgenden Felder fest:

    - **Land/Region**: Wählen Sie den Länder- bzw. Regionscode des Partners aus.
    - **Umsatzsteuernummer**: Geben Sie die Umsatzsteuernummer des Partners ein.
    - **Name der Firma**: Geben Sie den Namen des Partners ein.

13. Navigieren Sie zu **Debitoren** > **Debitoren** > **Alle Debitoren**, und befolgen Sie diese Schritte für jeden Debitor:

    1. Wählen Sie einen Debitor aus.
    2. Wählen Sie im **Rechnung und Lieferung**-Inforegister im Feld **Mehrwertsteuer** die **Umsatzsteuernummer** ausn und wählen Sie **Alle**.
    3. Die Umsatzsteuernummer des ausgewählten Debitors auswählen.

14. Navigieren Sie zu **Kreditorenkonten** > **Kreditoren** > **Alle Kreditoren**, und befolgen Sie diese Schritte für jeden Kreditor:

    1. Wählen Sie einen Kreditor aus.
    2. Wählen Sie im **Rechnung und Lieferung**-Inforegister im Feld **Mehrwertsteuer** die **Umsatzsteuernummer** ausn und wählen Sie **Alle**.
    3. Die Umsatzsteuernummer des ausgewählten Lieferanten auswählen.

15. Gehen Sie zu **Steuer** > **Einstellungen** > **Außenhandel** > **Komprimierung von Intrastat** und wählen Sie die Felder aus, die verglichen werden sollen, wenn Intrastat-Informationen zusammengefasst werden. Wählen Sie für Italien **Steuerbefreite Nummer**, **Transaktions-Code**, **Ware**, **Lieferbedingungen**, **Transport**, **Land/Region**, **Herkunftsland /-Region**, **Herkunftsland**, **Herkunfts-/Bestimmungsland**, **Währung**, **Monat**, **Quartal**, und **Korrekturjahr**.

## <a name="italian-vendor-invoice-journal-for-foreign-trade"></a>Italienisches Kreditorenrechnungsjournal für den Außenhandel

Wenn Sie mit dem Kreditorenrechnungsjournal arbeiten, müssen Sie unbedingt einige Parameter berücksichtigen, die sich auf Ihren Intrastat-Bericht auswirken. Auf der **Außenhandel** Registerkarte können Sie die folgenden italienspezifischen Felder überprüfen und festlegen.

| **Feld**                    | **Beschreibung**                                                                                 |
|------------------------------|-------------------------------------------------------------------------------------------------|
| Art des Geschäftes             | Wählen Sie den Buchungscode aus.                                                                    |
| Transport                    | Wählen Sie den Transport aus, der für die Übertragung verwendet wird.                                             |
| Warencode                    | Wählen Sie den Warencode des Artikels.                                                                 |
| Besondere Maßeinheit             | Geben Sie zusätzliche Einheiten ein, wenn für die ausgewählte Warennummer zusätzliche Einheiten gemeldet werden sollen. |
| Land/Region               | Wählen Sie das Land/die Region des Partners aus.                                                            |
| Landkreis für Ursprung/Ziel | Wählen Sie das Zielland aus.                                                                  |

## <a name="intrastat-journal"></a>Intrastat-Erfassungsfeld

Mithilfe der Intrastat-Erfassung können Sie Informationen zum Handel zwischen EU-Ländern/Regionen melden.

Um das Intrastat-Journal zu öffnen, gehen Sie zu **Steuer** &gt; **Erklärungen** &gt; **Außenhandel** &gt; **Intrastat**. Die folgenden Informationen werden für jede Transaktion angezeigt:

| **Feld**                              | **Beschreibung**                                                                                                                                                                                                                                                                                                                                                                                                         |
|----------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Datum                                   | Das Buchungsdatum der Buchung.                                                                                                                                                                                                                                                                                                                                                                                    |
| Planungsrichtung                              | Dieses Feld ist **Eingang** für Einkäufe oder **Versand** für Verkäufe eingestellt.                                                                                                                                                                                                                                                                                                                                            |

>
>Wenn Sie im gleichen Zeitraum wie der Berichtszeitraum eine negative Korrektur (Gutschrift) erhalten, müssen Sie das Intrastat-Journal wie folgt manuell ändern.
>
> 1. Wechseln Sie zu **Steuer** > **Meldungen** > **Außenhandel** > **Intrastat**.
> 2. Suchen und löschen Sie die als Korrektur gekennzeichnete Transaktion.
> 3. Suchen Sie die ursprüngliche Transaktion und ändern Sie den Wert des **Rechnungsbetrag** Feldes entsprechend.
> Angenommen, Sie erhalten von einem Kreditor eine Rechnung über 10.000 Euro und bezahlen -2.000 Euro. In diesem Fall müssen Sie das Intrastat-Journal öffnen und die Transaktion für -2.000 suchen und löschen. Suchen Sie dann die ursprüngliche Transaktion für 10.000 und setzen Sie den Rechnungsbetrag auf 8.000 (= 10.000 – 2.000).

### <a name="intrastat-transfer"></a>Intrastat-Übertragung

Im Aktivitätsbereich können Sie **Überweisen** auswählen, um die Informationen über den innergemeinschaftlichen Handel aus Ihren Verkaufsaufträgen, Freitextrechnungen, Bestellungen, Lieferantenrechnungen, Lieferantenprodukteingängen, Projektrechnungen und Transferaufträgen automatisch zu übertragen. Es werden nur Dokumente übermittelt, die ein EU-Land als Bestimmungsland oder -Region (bei Versendungen) haben.

Alternativ können Sie Transaktionen manuell eingeben, indem Sie **Neu** im Aktionsbereich auswählen.

Für jede Transaktion können Sie mehrere italienspezifische Parameter auf der Registerkarte **Allgemein** festlegen. Die folgende Tabelle enthält weitere Informationen zu den Feldern.

| Feld | Beschreibung |
|-------|-------------|
| Positionstyp | Dieses Feld kann eingestellt werden auf **Waren** oder **Dienstleistungen**.<br> Damit eine Transaktion als Dienstleistung betrachtet wird, sollte Ihr Rechnungsposten auf eine der folgenden Arten eingerichtet sein:<br>- Es hat keinen Warencode.<br>- Es hat einen sechsstelligen Warencode.  |
| Ursprungslandkreis | Der Ursprungsbezirk des Produkts oder der Dienstleistung. Dieser Wert wird auf der Seite **Freigegebene Produkte** angegeben. |

| Quartal | Das Quartal der ursprünglichen Transaktlion. |
| Jahr der Korrektur | Das Jahr der ursprünglichen Transaktion. |

### <a name="generate-an-intrastat-report"></a>Generieren eines Intrastat-Berichts

1. Um den Intrastat-Bericht zu generieren, gehen Sie zu **Steuer** > **Erklärungen** > **Außenhandel** > **Intrastat**.
2. Wählen Sie im Aktivitätsbereich **Ausgabe** > **Bericht**.
3. Wählen Sie im Dialogfeld **Intrastat-Bericht** die Start- und Enddaten für den Bericht aus.
4. Wählen Sie im Feld **Datei generieren** die Option **Ja** aus, um eine .txt.-Datei zu generieren und den Namen der Datei einzugeben.
5. Wählen Sie im Feld **Bericht generieren** die Option **Ja** aus, um eine .xlsx-Datei zu generieren und dann den Namen des Berichts einzugeben.
6. Wählen Sie **Eingänge** oder **Ausgänge** basierend auf der Art des Berichts aus.
7. Geben Sie im Feld **Referenznummer** die Belegnummer ein. Dieser Wert beeinflusst den Dateinummer Code auf dem Intrastat-Dateibericht.
