# <a name="post-fixed-asset-transactions-to-posting-layers"></a><span data-ttu-id="c472b-103">Buchen von Anlagenbuchungen auf Buchungsebenen</span><span class="sxs-lookup"><span data-stu-id="c472b-103">Post fixed asset transactions to posting layers</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="c472b-104">Dieser Artikel gibt eine Übersicht über das Buchen von Ebenenfunktionen für Anlagenbuchungen.</span><span class="sxs-lookup"><span data-stu-id="c472b-104">This article gives an overview of posting layer functionality for fixed asset transactions.</span></span>

<span data-ttu-id="c472b-105">Eine Anlage wird für unterschiedliche Zwecke häufig mit unterschiedlichen Methoden abgeschrieben.</span><span class="sxs-lookup"><span data-stu-id="c472b-105">A fixed asset is often depreciated in different ways for different purposes.</span></span> <span data-ttu-id="c472b-106">Die Abschreibung zu steuerlichen Zwecken wird anhand der aktuellen Steuerregeln berechnet, um die höchstmögliche Abschreibung vor Steuern zu erreichen, wohingegen die Abschreibung zu Berichtszwecken entsprechend den Buchhaltungsgesetzen und -standards berechnet wird.</span><span class="sxs-lookup"><span data-stu-id="c472b-106">Depreciation for tax purposes is calculated by using current tax rules to achieve the highest possible depreciation before taxes, but depreciation for reporting purposes is calculated according to accounting laws and standards.</span></span> <span data-ttu-id="c472b-107">Die unterschiedlichen Abschreibungsarten werden auf den Buchungsebenen separat berechnet und aufgezeichnet.</span><span class="sxs-lookup"><span data-stu-id="c472b-107">The various kinds of depreciation are calculated and recorded separately in the posting layers.</span></span>

<span data-ttu-id="c472b-108">Jedes Buch, das einer Anlage zugeordnet ist, wird für eine bestimmte Buchungsebene eingerichtet, die ein übergeordnetes Abschreibungsziel aufweist.</span><span class="sxs-lookup"><span data-stu-id="c472b-108">Each book that is attached to a fixed asset is set up for a particular posting layer that has an overall depreciation objective.</span></span> <span data-ttu-id="c472b-109">Es gibt zehn Buchungsebenen: Aktuelle, Vorgänge, Steuern und sieben benutzerdefinierte Ebenen.</span><span class="sxs-lookup"><span data-stu-id="c472b-109">There are ten posting layers: Current, Operations, Tax, and seven Custom layers.</span></span> <span data-ttu-id="c472b-110">Sie können Buchungen im Hauptbuch für das Buch deaktivieren, indem Sie das Feld Ins Hauptbuch buchen auf Nein festlegen.</span><span class="sxs-lookup"><span data-stu-id="c472b-110">You can also disable posting to the general ledger for the book by setting the Post to general ledger field to No.</span></span> <span data-ttu-id="c472b-111">Das Feld Buchungsebene wird dann automatisch auf Keine gesetzt.</span><span class="sxs-lookup"><span data-stu-id="c472b-111">The Posting layer field is then automatically set to None.</span></span> <span data-ttu-id="c472b-112">Bücher, die nicht im Hauptbuch buchen, werden in der Regel für die Steuererklärung verwendet.</span><span class="sxs-lookup"><span data-stu-id="c472b-112">Typically, books that don’t post to the general ledger are used for tax reporting purposes.</span></span> <span data-ttu-id="c472b-113">Dies bietet zusätzliche Flexibilität zur Löschung von historischen Buchungen für das Anlagenbuch, da sie nicht im Hauptbuch eingerichtet wurden.</span><span class="sxs-lookup"><span data-stu-id="c472b-113">This approach gives you the additional flexibility to delete historical transactions for the asset book, because they haven’t been committed to the general ledger.</span></span>

<span data-ttu-id="c472b-114">Anlagenerfassungen definieren sich durch die Nutzung der Seite  Journalnamen unter Hauptbuch > Journaleinrichtung > Journalnamen.</span><span class="sxs-lookup"><span data-stu-id="c472b-114">Fixed asset journals are defined by using the Journal names page at General ledger > Journal setup > Journal names.</span></span> <span data-ttu-id="c472b-115">Jede Erfassung, in der Sie Abschreibungen buchen können, wird anhand des Erfassungsnamens (Journal) für eine einzelne Buchungsebene definiert.</span><span class="sxs-lookup"><span data-stu-id="c472b-115">Each journal that you can post depreciations in is defined by its journal name for only one posting layer.</span></span> <span data-ttu-id="c472b-116">Die Buchungsebene kann im Journal nicht geändert werden.</span><span class="sxs-lookup"><span data-stu-id="c472b-116">The posting layer in the journal can’t be changed.</span></span> <span data-ttu-id="c472b-117">Diese Einschränkung hilft sicherzustellen, dass Buchungen für die einzelnen Buchungsebenen getrennt gehalten werden.</span><span class="sxs-lookup"><span data-stu-id="c472b-117">This restriction helps guarantee that transactions for each posting layer are kept separate.</span></span> <span data-ttu-id="c472b-118">Für jede Buchungsebene muss mindestens ein Journal angelegt werden.</span><span class="sxs-lookup"><span data-stu-id="c472b-118">At least one journal name must be created for each posting layer.</span></span> <span data-ttu-id="c472b-119">Auf wenn Sie Bücher verwenden, die nicht auf das Sachkonto buchen, müssen Sie ein Journal erstellen, bei dem die Buchungsebene auf Kein festgelegt wird.</span><span class="sxs-lookup"><span data-stu-id="c472b-119">If you’re using books that don’t post to the general ledger, you must also create a journal where the posting layer is set to None.</span></span>

<span data-ttu-id="c472b-120">Sie können Sachkonten für Anlagenbuchungen auf der Seite Anlagenbuchungsprofile festlegen.</span><span class="sxs-lookup"><span data-stu-id="c472b-120">You can designate ledger accounts for fixed asset transactions on the Fixed asset posting profiles page.</span></span> <span data-ttu-id="c472b-121">Sie müssen für jedes Buchungsprofil die relevante Buchungsart und das Buch auswählen und können dann die Sachkonten angeben.</span><span class="sxs-lookup"><span data-stu-id="c472b-121">For each posting profile, you must select the relevant transaction type and book, and then designate the ledger accounts.</span></span> <span data-ttu-id="c472b-122">Einrichten eines Buchungsprofildatensatz für jedes Buch, das im Hauptbuch gebucht wird.</span><span class="sxs-lookup"><span data-stu-id="c472b-122">Set up a posting profile record for each book that will post to the general ledger.</span></span>

<span data-ttu-id="c472b-123">Die Anlage kann in Dokumente eingetragen werden, die nur die **Aktuell**-Buchungsebene unterstützen, z. B. **Bestellung**, **Ausstehende Kreditorenrechnung**, **Auftrag** oder **Freitextrechnung**.</span><span class="sxs-lookup"><span data-stu-id="c472b-123">The fixed asset can be entered in documents that only support the **Current** posting layer, like **Purchase order**, **Pending vendor invoice**, **Sales order**, or **Free text invoice**.</span></span> <span data-ttu-id="c472b-124">Bei der Auswahl einer Anlagen-ID in einem dieser Dokumente wird das Anlagenbuch in das Buch mit der **Aktuell**-Buchungsebene gefiltert und während der Buchung automatisch ausgefüllt, wenn das System überprüft, ob die Buchungsebene für Anlagen handelt **Aktuell** ist.</span><span class="sxs-lookup"><span data-stu-id="c472b-124">While selecting a fixed asset ID in any of those documents the asset book is filtered to the book with **Current** posting layer, and will be filled in automatically, during posting when the system validates that the fixed asset posting layer is **Current**.</span></span> <span data-ttu-id="c472b-125">Wenn diese Validierung nicht abgeschlossen werden kann, wird der Buchungsvorgang gestoppt.</span><span class="sxs-lookup"><span data-stu-id="c472b-125">If that validation can't be completed, the posting process will be stopped.</span></span> 

> [!NOTE] 
> <span data-ttu-id="c472b-126">Bei Verwendung von abgeleiteten Büchern können Buchungen gleichzeitig auf unterschiedlichen Buchungsebenen durchgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="c472b-126">By using derived books, you can post transactions to different posting layers at the same time.</span></span> <span data-ttu-id="c472b-127">Die Transaktionen dieses primären Buchs werden in einer Erfassung oder einem Quelldokument erstellt, in dem die Buchungsebene der Buchungsebene des Buchs entspricht.</span><span class="sxs-lookup"><span data-stu-id="c472b-127">The transactions of the primary book are created in a journal or source document where the posting layer corresponds to the book posting layer.</span></span> <span data-ttu-id="c472b-128">Beim Buchen werden die Buchungen der abgeleiteten Buchtransaktionen dann auf den entsprechenden Buchungsebenen gebucht.</span><span class="sxs-lookup"><span data-stu-id="c472b-128">During posting, the derived book transactions will be posted to the appropriate posting layers.</span></span>
