# <a name="manage-standard-cost-updates"></a><span data-ttu-id="75cab-103">Standardkostenaktualisierungen verwalten</span><span class="sxs-lookup"><span data-stu-id="75cab-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="75cab-104">Die Aktualisierung von Standardkostendaten kann auf zwei unterschiedliche Arten verwaltet werden - durch den Einzelversionsansatz oder durch den Zwei-Versionen-Ansatz.</span><span class="sxs-lookup"><span data-stu-id="75cab-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span>

<span data-ttu-id="75cab-105">Beim Einzelversionsansatz wird eine Nachkalkulationsversion mit sämtlichen Kostendatensätzen verwendet.</span><span class="sxs-lookup"><span data-stu-id="75cab-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="75cab-106">Diese Datensätze beinhalten die anfänglich definierten Kosten sowie allen Kostenaktualisierungen.</span><span class="sxs-lookup"><span data-stu-id="75cab-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="75cab-107">Beim Zwei-Versionen-Ansatz enthält eine Version die Datensätze der anfänglich definierten Kosten und eine zweite Version alle Datensätze der Kostenaktualisierungen.</span><span class="sxs-lookup"><span data-stu-id="75cab-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="75cab-108">Einer der wichtigsten Vorteile des Zwei-Versionen-Ansatzes besteht in der übersichtlichen Darstellung sowie in der Nachverfolgung von Kostenaktualisierungen in einer eigenen Nachkalkulationsversion, ohne dass sich dies auf die ursprüngliche Nachkalkulationsversion auswirkt.</span><span class="sxs-lookup"><span data-stu-id="75cab-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="75cab-109">Der Zwei-Versionen-Ansatz kann zur Erkennung mehrerer stufenweiser Aktualisierungen verwendet werden, bei denen jede einzelne stufenweise Aktualisierung eine eigene Nachkalkulationsversion mit den stufenweisen Kostendatensätzen besitzt.</span><span class="sxs-lookup"><span data-stu-id="75cab-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>
