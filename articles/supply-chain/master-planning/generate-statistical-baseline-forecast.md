# <a name="generate-a-statistical-baseline-forecast"></a><span data-ttu-id="8ec58-103">Statistische Grundplanung generieren</span><span class="sxs-lookup"><span data-stu-id="8ec58-103">Generate a statistical baseline forecast</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="8ec58-104">Dieser Artikel enthält Informationen zu den Parametern und Filtern, die in der Berechnung der Bedarfsplanung verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-104">This article provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="8ec58-105">Wenn Sie eine Grundplanung erstellen, müssen Sie zunächst die Parameter und die Filter angeben, die bei der Berechnung verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="8ec58-106">So können Sie z. B. eine Grundplanung erstellen, die Bedarf basierend auf Buchungsdaten vom letzten Jahr für ein bestimmtes Unternehmen, für den kommenden Monat und für eine ausgewählte Gruppe von Artikeln einschätzt.</span><span class="sxs-lookup"><span data-stu-id="8ec58-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="8ec58-107">Um eine Bedarfsplanung zu generieren, wechseln Sie zu **Produktprogrammplan &gt;Planung &gt; Bedarfsplanung &gt;Statistische Grundplanung generieren**.</span><span class="sxs-lookup"><span data-stu-id="8ec58-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="8ec58-108">Der Planungszeitrahmen kann zum Zeitpunkt der Planunsgenerierung ausgewählt werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="8ec58-109">Die verfügbaren Werte sind Tag, Woche und Monat.</span><span class="sxs-lookup"><span data-stu-id="8ec58-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="8ec58-110">Die Anzahl von Planungszeitrahmen, für die die Planung generiert wird, wird im Feld **Planungshorizont** festgelegt.</span><span class="sxs-lookup"><span data-stu-id="8ec58-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="8ec58-111">Wenn die Planungsstrategie auf **Über historischen Bedarf kopieren** festgelegt ist, wird das Ende des historischen Horizonts ignoriert.</span><span class="sxs-lookup"><span data-stu-id="8ec58-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="8ec58-112">Das System kopiert die Anzahl der Zeitrahmen, die im Feld **Planungshorizont** angegeben ist, in den Planungsbedarf, beginnend ab dem Startdatum im Feld **Von Datum** unter **Historischer Horizont**.</span><span class="sxs-lookup"><span data-stu-id="8ec58-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="8ec58-113">Indem historischer Bedarf von einem bestimmten Datum in ein zukünftiges Datum kopiert wird, können Produktionsplaner den Plan für die nächsten Quartale auf zwei Arten erstellen:</span><span class="sxs-lookup"><span data-stu-id="8ec58-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="8ec58-114">Durch Kopieren des Bedarfs aus demselben Quartal im letzten Jahr</span><span class="sxs-lookup"><span data-stu-id="8ec58-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="8ec58-115">Durch Kopieren des Bedarfs aus vorherigen Quartal</span><span class="sxs-lookup"><span data-stu-id="8ec58-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="8ec58-116">Um Unklarheiten in den Produktionsplänen zu verhindern, können mehrere Planungszeitrahmen eingefroren werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="8ec58-117">Diese Zahl wird im Feld **Nichtplanungszeitraum** festgelegt.</span><span class="sxs-lookup"><span data-stu-id="8ec58-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="8ec58-118">Auf der Seite **Angepasste Bedarfsplanung** werden die Zellen für die fixierten Zeitrahmen deaktiviert, um optisch anzugeben, dass diese Werte nicht geändert werden sollen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="8ec58-119">Das Startdatum für die Grundbedarfsplanung muss nicht das aktuelle Datum oder ein Datum in der Zukunft sein.</span><span class="sxs-lookup"><span data-stu-id="8ec58-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="8ec58-120">Um ein anderes Startdatum festzulegen, verwenden Sie das Feld **Startdatum für statistische Grundplanung - Von Datum**.</span><span class="sxs-lookup"><span data-stu-id="8ec58-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="8ec58-121">So können Benutzer im Juni beispielswseise eine Planung für das nächste Jahr generieren.</span><span class="sxs-lookup"><span data-stu-id="8ec58-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="8ec58-122">Da die Planungszeitrahmen zwischen dem Ende des historischen Bedarfs und dem Beginn der Grundbedarfs fehlen, sind die Vorhersagen nicht genau.</span><span class="sxs-lookup"><span data-stu-id="8ec58-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="8ec58-123">Wenn Sie den Microsoft Dynamics 365 for Finance and Operations -Bedarfsplanungsdienst verwenden, gibt es vier Arten, auf die Sie die fehlenden Lücken ausfüllen können.</span><span class="sxs-lookup"><span data-stu-id="8ec58-123">If you are using the Microsoft Dynamics 365 for Finance and Operations Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="8ec58-124">Sie können die Methode auswählen, die Sie wünschen, indem Sie den MISSING\_VALUE\_SUBSTITUTION-Parameter auf der Seite **Bedarfsplanungsparameter** festlegen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

<span data-ttu-id="8ec58-125">Das Feld **Startdatum für statistische Grundplanung** - **Von Datum** muss auf den Beginn eines Planungszeitrahmens festgelegt werden, beispielsweise in den USA ein Sonntag, wenn der Planungszeitrahmen die Woche ist.</span><span class="sxs-lookup"><span data-stu-id="8ec58-125">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="8ec58-126">Das System reguliert automatisch das Feld **Startdatum für statistische Grundplanung** - **Von Datum**, um den Beginn eines Planungszeitrahmens abzugleichen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-126">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="8ec58-127">Das Feld **Startdatum für statistische Grundplanung** - **Von Datum** kann auf ein Datum in der Vergangenheit gesetzt werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-127">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="8ec58-128">Das bedeutet, es ist möglich, eine Bedarfsplanung in der Vergangenheit zu generieren.</span><span class="sxs-lookup"><span data-stu-id="8ec58-128">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="8ec58-129">Dies ist hilfreich, da Benutzer die Planungsdienstparameter ändern können, damit die statistische Planung, die in der Vergangenheit generiert wurde, mit dem tatsächlichen historischen Bedarf übereinstimmt.</span><span class="sxs-lookup"><span data-stu-id="8ec58-129">This is useful, because it lets users tweak the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="8ec58-130">Benutzer können mithilfe dieser Parametereinstellungen dann fortfahren, um eine statistische Grundplanung für die Zukunft zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-130">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="8ec58-131">Die manuellen Anpassungen, die in den vorherigen Bedarfsplanungsiterationen vorgenommen wurden, können automatisch in die neue Grundplanung übernommen werden, wenn das Kontrollkästchen **Manuelle Anpassungen auf Bedarfsplanung übertragen** aktiviert ist.</span><span class="sxs-lookup"><span data-stu-id="8ec58-131">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas** t check box is selected.</span></span> <span data-ttu-id="8ec58-132">Wenn das Kontrollkästchen deaktiviert ist, werden die manuellen Anpassungen nicht zur Grundplanung hinzugefügt, sie werden jedoch nicht gelöscht.</span><span class="sxs-lookup"><span data-stu-id="8ec58-132">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="8ec58-133">Die manuellen Anpassungen, die an einer Planung vorgenommen werden, können nur zum Zeitpunkt des Planungsimports gelöscht werden, indem das Kontrollkästchen **Manuelle Anpassungen der Grundbedarfsplanung speichern** deaktiviert wird.</span><span class="sxs-lookup"><span data-stu-id="8ec58-133">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="8ec58-134">Manuelle Anpassungen werden zum Zeitpunkt der Autorisierung gespeichert.</span><span class="sxs-lookup"><span data-stu-id="8ec58-134">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="8ec58-135">Wenn ein Benutzer manuelle Anpassungen an der Planung vornimmt, aber die Planung nicht in Dynamics 365 for Finance and Operations autorisiert, gehen die Änderungen daher verloren.</span><span class="sxs-lookup"><span data-stu-id="8ec58-135">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Finance and Operations, the changes are lost.</span></span> <span data-ttu-id="8ec58-136">Weitere Informationen zu manuellen Anpassungen und dazu, wie sie arbeiten, finden Sie unter [Autorisieren der angepassten Planung](authorize-adjusted-forecast.md).</span><span class="sxs-lookup"><span data-stu-id="8ec58-136">For more information about manual adjustments and how they work, see [Authorizing the adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="8ec58-137">Eine Bedarfsplanungsgenerierung kann einen Namen und Kommentare haben, damit Benutzer die Planung erkennen können, die generiert wurde.</span><span class="sxs-lookup"><span data-stu-id="8ec58-137">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="8ec58-138">Diese Werte werden im Generierungsverlauf der Planung auf der Seite **Generierungsverlauf statistische Grundplanung** angezeigt.</span><span class="sxs-lookup"><span data-stu-id="8ec58-138">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="8ec58-139">Die Intercompany-Planungsgruppe, die Artikelverteilungsschlüssel und andere Filter können zum Zeitpunkt der Planungsgenerierung angewendet werden.</span><span class="sxs-lookup"><span data-stu-id="8ec58-139">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="8ec58-140">Sie können verwendet werden, um die Leistung zu verbessern oder die Daten in überschaubare Blöcke aufzuteilen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-140">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="8ec58-141">Beachten Sie jedoch, dass keine Bedarfsplanung für die Mitglieder eines Artikelverteilungsschlüssels generiert wird, der keiner Intercompany-Planungsgruppe zugeordnet ist, auch wenn der Artikelverteilungsschlüssel in der Abfrage ausgewählt wird.</span><span class="sxs-lookup"><span data-stu-id="8ec58-141">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span> 

<span data-ttu-id="8ec58-142">**Tipp**: Manchmal erhalten Benutzer Fehler, während sie eine Bedarfsplanung generieren oder die Planungsgenerierung wird ohne Sitzungsprotokoll abgeschlossen.</span><span class="sxs-lookup"><span data-stu-id="8ec58-142">**Tip**: Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="8ec58-143">Dies kann aufgrund der übrigen Daten in der Abfrage auftreten, die zuvor für die Planungsgenerierung verwendet wurde.</span><span class="sxs-lookup"><span data-stu-id="8ec58-143">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="8ec58-144">Um dieses Problem zu beheben, klicken Sie auf **Auswählen**, um die Seite **Abfrage** zu öffnen, klicken Sie auf **Zurücksetzen**, und generieren Sie die Grundplanung neu.</span><span class="sxs-lookup"><span data-stu-id="8ec58-144">To fix this issue, click **Select** to open the **Query** page, click **Reset**, and then regenerate the baseline forecast.</span></span> 

<span data-ttu-id="8ec58-145">Wenn die Planung nicht für einen großen Satz Artikel generiert wird, sondern beispielsweise jeweils für einen Artikel oder einen Artikelverteilungsschlüssel, dann können Sie für eine bessere Leistung das Kontrollkästchen **Anforderungsantwortmodus verwenden** auf der Registerkarte **Produktprogrammplan - Einstellungen - Bedarfsplanung**  -  **Bedarfsplanungsparameter - Azure Machine Learning** aktivieren.</span><span class="sxs-lookup"><span data-stu-id="8ec58-145">If the forecast is not generated for a big set of items, but, for example, for one item or one item allocation key at a time, then in order to get better performance, you can select the **Use request response mode** check box on the **Master planning - Setup - Demand forecasting** - **Demand forecasting parameters - Azure Machine Learning** tab.</span></span>
