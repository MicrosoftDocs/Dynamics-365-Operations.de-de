# <a name="reason-codes-for-service-orders"></a>Ursachencodes für Serviceaufträge   

[!include [banner](../includes/banner.md)]


Ursachencodes können Sie verwenden, um den Status eines Serviceauftrags besser zu erläutern, wenn die Phase eines Serviceauftrags aktualisiert wird. Wenn Sie beispielsweise einen Serviceauftrag stornieren, können Sie einen Ursachencode für die Stornierung auswählen.

Führen Sie zum Anzeigen von Informationen über Ursachencodes, die verwendet werden, um den Fortschritt von Serviceaufträgen zu verfolgen, den Bericht „Serviceauftragsverlauf“ aus. In diesem Bericht werden – unabhängig vom jeweiligen Status – alle Serviceaufträge sowie die Ursachencodes aufgeführt, die beim Aktualisieren der Serviceauftragsphase angegeben werden.

## <a name="turn-reason-codes-on-or-off"></a>Aktivieren und Deaktivieren von Ursachencodes

Die Verwendung von Ursachencodes ist optional. Sie können entscheiden, ob ein Ursachencode erforderlich ist, wenn Sie einen Serviceauftrag zu einer bestimmten Servicephase aktualisieren.

1.  Klicken auf **Serviceverwaltung** \> **Einrichtung** \> **Serviceaufträge** \> **Servicephasen**.

2.  Wählen Sie im Formular **Servicephasen** eine Servicephase aus, und aktivieren Sie anschließend das Kontrollkästchen **Ursache** für die Servicephase.

3.  Schließen Sie das Formular, um Ihre Änderungen zu speichern.
