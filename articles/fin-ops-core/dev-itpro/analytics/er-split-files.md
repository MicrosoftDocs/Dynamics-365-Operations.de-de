<span data-ttu-id="6aad9-104">Sie können Elektronische Berichterstellung(ER)-Formate entwerfen, um ausgehende Dokumente im XML-Format zu generieren.</span><span class="sxs-lookup"><span data-stu-id="6aad9-104">You can design Electronic reporting (ER) formats to generate outgoing documents in XML format.</span></span> <span data-ttu-id="6aad9-105">Manchmal können diese Dokumente nur akzeptiert werden, wenn sie bestimmte Kriterien erfüllen, wie z.B. eine maximale Dateigröße oder eine maximale Anzahl von XML-Knoten.</span><span class="sxs-lookup"><span data-stu-id="6aad9-105">Sometimes, those documents can be accepted only when they meet specific criteria, such a maximum file size or a maximum number of some XML nodes.</span></span> <span data-ttu-id="6aad9-106">Sie können ER-Formate entwerfen, um elektronische Dokumente zu erzeugen, die den Anforderungen der Empfänger dieser Dokumente entsprechen.</span><span class="sxs-lookup"><span data-stu-id="6aad9-106">You can design ER formats to generate electronic documents that satisfy the requirements that the recipients of those documents specify.</span></span>

- <span data-ttu-id="6aad9-107">Für das Formatelement FILE können Sie eine Begrenzung der Dateigröße als ER-Ausdruck definieren.</span><span class="sxs-lookup"><span data-stu-id="6aad9-107">For the FILE format element, you can define a limit on the file size as an ER expression.</span></span> <span data-ttu-id="6aad9-108">Wenn die definierte Grenze bei der Erstellung eines ER-Berichts überschritten wird, beendet ER die Erstellung der aktuellen Datei und fährt dann mit der Erstellung der nächsten Datei fort.</span><span class="sxs-lookup"><span data-stu-id="6aad9-108">If the defined limit is exceeded when an ER report is generated, ER finishes creating the current file and then moves on to create the next file.</span></span>

- <span data-ttu-id="6aad9-111">Für jedes XML SEQUENCE-Formatelement können Sie eine Begrenzung der Anzahl der untergeordneten Elemente als ER-Ausdruck definieren.</span><span class="sxs-lookup"><span data-stu-id="6aad9-111">For any XML SEQUENCE format element, you can define a limit on the number of child elements as an ER expression.</span></span> <span data-ttu-id="6aad9-112">Wenn die Anzahl der geschachtelten XML-Knoten des Formatelements in der generierten Datei die definierte Grenze überschreitet, wenn ein ER-Bericht ausgeführt wird, beendet ER die Erstellung der aktuellen Datei und fährt dann mit der Erstellung der nächsten Datei fort.</span><span class="sxs-lookup"><span data-stu-id="6aad9-112">If the number of nested XML nodes of the format element in the generated file exceeds the defined limit when an ER report is run, ER finishes creating the current file and then moves on to create the next file.</span></span>
- <span data-ttu-id="6aad9-113">Sie können jedes Formatelement vom Typ XML ELEMENT als non-breakable festlegen.</span><span class="sxs-lookup"><span data-stu-id="6aad9-113">You can mark any XML ELEMENT format element as non-breakable.</span></span> <span data-ttu-id="6aad9-114">Auf diese Weise können Sie die verschachtelten Elemente von XML-Knoten, die unter dem Formatelement erzeugt werden, in einer einzigen generierten Datei behalten.</span><span class="sxs-lookup"><span data-stu-id="6aad9-114">In this way, you can keep the nested items of XML nodes that are generated under the format element in a single generated file.</span></span>

<span data-ttu-id="6aad9-115">Zusätzlich zu den Formatelementen XML ELEMENT und XML SEQUENCE zum Hinzufügen von XML-Knoten in die generierte Datei, können Sie das RAW XML Formatelement verwenden.</span><span class="sxs-lookup"><span data-stu-id="6aad9-115">In addition to using the XML ELEMENT and XML SEQUENCE format elements to add XML nodes to the generated file, you can use the RAW XML format element.</span></span> <span data-ttu-id="6aad9-116">Knoten, die Sie mit dem RAW XML Formatelement hinzufügen, werden jedoch nicht berücksichtigt, wenn die Anzahl der Knoten berechnet wird, um die Grenzen der Anzahl der Elemente auszuwerten.</span><span class="sxs-lookup"><span data-stu-id="6aad9-116">However, nodes that you add by using the RAW XML format element aren't considered when the number of nodes is calculated to evaluate the limits on the number of elements.</span></span>

<span data-ttu-id="6aad9-117">Wenn Sie Dateiziele für ein FILE Formatelement konfiguriert haben, das so konfiguriert wurde, dass es die generierte Ausgabe aufteilt, wenn bestimmte Grenzwerte überschritten werden, wird jedes Stück der generierten Ausgabe als einzelne Datei an das konfigurierte Dateiziel gesendet.</span><span class="sxs-lookup"><span data-stu-id="6aad9-117">If you configured file destinations for a FILE format element that has been configured to split the generated output whenever specific limits are exceeded, each piece of generated output is sent to the configured file destination as an individual file.</span></span> <span data-ttu-id="6aad9-118">Um die Dateien, die durch die Aufteilung der Ausgabe erzeugt werden, eindeutig zu benennen, müssen Sie einen ER-Ausdruck für das FILE Formatelement konfigurieren.</span><span class="sxs-lookup"><span data-stu-id="6aad9-118">To uniquely name the files that are created by splitting the output, you must configure an ER expression for the FILE format element.</span></span> <span data-ttu-id="6aad9-119">Wenn Sie eine ER-Datenquelle vom Typ NUMBER SEQUENCE einbinden, wird die Zahlenfolge für jedes Stück der aufgeteilten Ausgabe inkrementiert.</span><span class="sxs-lookup"><span data-stu-id="6aad9-119">If you include an ER data source of the NUMBER SEQUENCE type, the number sequence will be incremented for each piece of the split output.</span></span>
